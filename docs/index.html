From Video to Fingering

Three Steps, One Goal

Video → MIDI → Calibration → Hand Tracking → Fingering → Bundle

  1. 1

    Video → MIDI

    Have a performance video but no MIDI? No problem. SMaPE uses Kong AI to transcribe piano music directly from audio, capturing pitch, timing, velocity, and sustain pedal data.

    Or: Supply your own MIDI for pixel-perfect alignment.

  2. 2

    Calibrate the Keyboard

    Click 4+ keys (C and G are easiest) to teach SMaPE where the keyboard is in your video. Handles any camera angle, perspective, or zoom.

    Result: A 1-D projective map from pixels to MIDI pitches.

  3. 3

    Track Hands & Detect Positions

    MediaPipe detects hand landmarks frame-by-frame. For each MIDI note, SMaPE finds the nearest fingertip and assigns hand (L/R) and finger (1-5).

    Result: Hand and finger position for every note, with confidence scores.

The Magic

Under the Hood

MIDI: Transcribed or Supplied

If you have MIDI: Use it directly. Extract video audio, find the first note onset with librosa, calculate the video-to-MIDI offset.

If you don't: Kong piano model analyzes the video's audio in parallel with video processing to generate MIDI with full note information. Offset is naturally 0 since MIDI comes from the same audio.

Projective Keyboard Mapping

Your calibration clicks (4+ keys) define a 1-D projective row map from screen pixels to MIDI pitches. This handles camera perspective, tilt, and zoom without needing explicit camera parameters.

Why not 4 corners? Corner clicks can't constrain depth, and they need you to guess which pitches sit at the ends. Real key clicks anchor actual pitches directly to pixels.

Parallel Hand Tracking

MediaPipe's HandLandmarker detects 21 landmark points per hand at 30 FPS, with 3D coordinates and confidence scores. This runs in parallel with MIDI transcription for speed.

Each frame captures the full hand pose—not just fingertip positions.

Depth-Invariant Matching

For each MIDI note onset, interpolate fingertip positions to that exact time. Convert both note pitch and fingertips to a continuous "white-key index"—matching along the keyboard (left-right), not in raw pixels.

Why? Raw pixel matching penalizes natural depth spread of fingers. White-key matching is what matters for playing.

Confidence Scoring

Distance between fingertip and expected key decays confidence (0–1). A match within half a key scores >0.5; one key off scores ~0.3.

Chord handling groups simultaneous notes and resolves finger conflicts with greedy nearest-first assignment.

The Finishing Touch

The .symple Bundle & Symplethesia

Everything you need in one file.

What's Inside a .symple Bundle

A .symple file is a ZIP archive containing:

  • manifest.json – format version, metadata (artist, title, genre, difficulty), source info
  • song.mid – the exact MIDI used for analysis (with sustain pedal CC64)
  • fingering.json – hand and finger assignments
  • DISCLAIMER.txt – accuracy notes, tools used, feedback link

ZIP is intentionally simple—no special tools, no dependencies. Both reader and writer get battle-tested stdlib support.

{ "version": 1, "generator": "piano-fingering", "metadata": { "artist": "Erik Satie", "title": "Gymnopédie No. 1", "genre": "Classical", "difficulty": "intermediate" }, "contents": [ "song.mid", "fingering.json", "DISCLAIMER.txt" ] }
One-step import: Open Symplethesia → File → Open → select the .symple file. The app imports the bundled MIDI, applies the fingering, and pre-fills metadata (artist, title, difficulty) automatically. No separate steps, no manual work.

How Symplethesia Reads It

The fingering importer joins each SMaPE note to a project note by:

  • Exact pitch match
  • Nearest note start within PPQ/8 ticks
  • Hand + finger applied in two reversible undo steps

You get full control over the result, and every change is undoable.

Get Started

Three Minutes to Your First Fingering

Install

python3 -m venv .venv source .venv/bin/activate pip install -r requirements.txt

Run the GUI

python3 gui.py

Or Command-Line

python3 extract_fingering.py --video performance.mp4 --midi performance.mid --out fingering.json
No MIDI file? Add --transcribe to generate it from the video's audio automatically.
