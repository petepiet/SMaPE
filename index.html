SMaPE

Extract piano fingering from videos in seconds. Generate MIDI, detect hand and finger positions, and create shareable bundles for Symplethesia.

Get Started on GitHub Support the Project

How It Works

1. Video → MIDI (Auto-generated or supplied)

Have a piano performance video but no MIDI file? SMaPE generates one automatically using advanced AI transcription (Kong piano model). Or supply your own MIDI for pixel-perfect alignment.

  • Kong AMT model detects pitch, timing, velocity, and pedal
  • Works with YouTube links or local files
  • Synthesia render support (keyboard videos)

2. Calibrate the Keyboard

Click 4+ keys (C and G are easiest) to teach SMaPE where the keyboard is in your video. The tool uses projective geometry to map video pixels to exact MIDI pitches.

  • Interactive calibration with live preview
  • Works with overhead and angled cameras
  • Handles any keyboard position or perspective

3. Track Hands & Extract Fingering

MediaPipe detects hand landmarks in real-time. For each MIDI note onset, SMaPE finds the nearest fingertip and assigns the hand (L/R) and finger (1-5).

  • Per-frame hand tracking at 30 FPS
  • Confidence scoring for each match
  • Handles held notes, chords, and fast passages

The Magic Under the Hood

1

MIDI: Transcribed or Supplied

Supplied MIDI: Use directly, calculate offset from audio (librosa onset detection).
Auto-transcribed: Kong piano model analyzes the video's audio in parallel with video processing to generate MIDI with pitch, timing, velocity, and sustain-pedal data. Offset is naturally 0 since MIDI comes from the same audio.

2

Projective Keyboard Mapping

Your calibration clicks define a 1-D projective row map from screen pixels to MIDI pitches. This handles camera perspective, tilt, and zoom—no need to know exact camera parameters.

3

Parallel Hand Tracking

While MIDI is being transcribed (or immediately if MIDI is supplied), MediaPipe tracks hand landmarks frame-by-frame at 30 FPS. Each frame captures 21 landmark points per hand with 3D coordinates and confidence scores.

4

Depth-Invariant Matching

For each MIDI note onset, interpolate fingertip positions to that exact time. Convert both note pitch and fingertips to a continuous "white-key index"—matching along the keyboard (left-right), not in raw pixels (which penalizes natural depth spread of fingers).

5

Confidence Scoring

Distance between fingertip and expected key decays confidence (0–1). A match within half a key scores >0.5; one key off scores ~0.3. Chord handling groups simultaneous notes and resolves finger conflicts with greedy nearest-first assignment.

The .symple Bundle & Symplethesia Integration

What's Inside a .symple Bundle

A .symple file is a ZIP archive containing:

  • manifest.json – format version, metadata (artist, title, genre, difficulty), source info
  • song.mid – the MIDI used for analysis (with sustain pedal CC64)
  • fingering.json – hand and finger assignments
  • DISCLAIMER.txt – accuracy notes, tools used, feedback link

The ZIP format is deliberately simple: no special tools needed, readers are mature and tested.

manifest.json Example

{ "version": 1, "generator": "piano-fingering-tool", "createdAt": "2026-07-05T...", "source": { "video": "https://youtube.com/watch?v=...", "midi": "performance.mid" }, "metadata": { "artist": "Adele", "title": "Someone Like You", "genre": "Pop", "difficulty": "intermediate" }, "contents": [ "song.mid", "fingering.json", "DISCLAIMER.txt" ] }

Importing into Symplethesia

One-step import: Open Symplethesia → File → Open → select the .symple file. The app imports the bundled MIDI, applies the fingering, and pre-fills metadata (artist, title, difficulty) automatically.

The fingering importer in Symplethesia joins each SMaPE note to a project note by:

  • Exact pitch match
  • Nearest note start within PPQ/8 ticks
  • Hand + finger applied in two undo steps (reversible)

No separate steps needed: no manual MIDI import, no separate fingering picker, no metadata entry. Everything is bundled and ready to play.

Get Started in Minutes

Install

python3 -m venv .venv source .venv/bin/activate pip install -r requirements.txt

Run the GUI

python3 gui.py

Or Command-Line

python3 extract_fingering.py --video performance.mp4 --midi performance.mid --out fingering.json
No MIDI? Add --transcribe to generate it from the video's audio.
